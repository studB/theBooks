# 파일 PDF 내보내기 — 마크다운 렌더 결과 프린트

에디터 상단 바의 'PDF 내보내기' 버튼으로 현재 파일의 마크다운을 숨겨진 iframe에 렌더하고 window.print()를 띄워 사용자가 PDF로 저장하게 한다.

## Version

- Current: 0.15.0
- Bump: minor
- Target: 0.16.0

## Purpose

작성한 글을 앱 밖으로 공유·인쇄·보관하려면 PDF가 가장 보편적인 포맷이다. 별도 도구로 복사·변환하지 않고, 에디터에서 본 마진/서식 그대로 한 번에 PDF로 뽑을 수 있게 해 글쓰기 결과물의 마지막 단계를 앱 안에서 완결시킨다.

## Scope

**In scope**
- 에디터 상단 바에 'PDF 내보내기' 버튼 1개 추가 (현재 열린 파일 대상)
- 클릭 시 숨겨진 iframe을 만들어 현재 파일의 마크다운을 렌더 — ReferencePane과 동일하게 `react-markdown` + `remark-gfm` + `remark-breaks`
- iframe 문서에 print 전용 스타일 주입: `@page { size: A4; margin: <top> <right> <bottom> <left>; }` — 값은 해당 파일의 `margins`(cm) 재사용
- 본문 타이포(제목/단락/목록/표/인용/코드블록)를 읽기 좋은 인쇄용 스타일로 지정, 문서 제목(`file.name`)을 본문 상단에 1회 노출
- 렌더 완료 후 `iframe.contentWindow.print()` 호출 → 사용자가 OS 프린트 다이얼로그에서 'PDF로 저장' 선택
- `afterprint`(또는 타임아웃 fallback) 시 iframe 정리

**Out of scope (later)**
- 다이얼로그 없이 직접 `.pdf` 파일로 저장 (Rust/headless 렌더)
- raw 마크다운 텍스트 모드 출력
- 페이지 번호·머리말/꼬리말·표지
- 마크다운 내 이미지 경로 해석 (로컬/S3 상대경로 → PDF 임베드)
- 여러 파일 일괄 내보내기, 폴더 단위 내보내기
- 분할(split)/참조 패널 내용 포함
- 사용자별 폰트 임베드, 출력 테마 선택 UI

## Stages

1. **PDF 렌더 유틸** — `src/`에 `exportPdf(file)` 함수 또는 `usePdfExport` 훅 작성: 숨겨진 iframe 생성 → print용 HTML/CSS 주입 → `react-dom/client`의 `createRoot`로 iframe body에 마크다운 컴포넌트 렌더
2. **프린트 트리거 + 정리** — 렌더 안정화(rAF 또는 onLoad) 후 `contentWindow.focus()` + `print()`, `afterprint` 리스너로 root unmount + iframe 제거. 빈 본문/세션 없음 등 가드
3. **버튼 연결 + 인쇄 스타일** — `Editor.jsx` 상단 바에 'PDF 내보내기' 버튼 추가해 유틸 호출. `@page` 마진과 본문 인쇄 CSS 다듬기
4. **검증** — `bun tauri dev`로 ① 버튼 클릭 → 프린트 다이얼로그 ② PDF 저장 결과에 사이드바/툴바 안 섞임 ③ 마진이 에디터와 일치 ④ GFM 표/목록/코드블록/한글 정상 ⑤ 빈 문서·긴 문서(다중 페이지) 확인

## Constraints

- 신규 npm 패키지 없음 — `react-markdown`/`remark-gfm`/`remark-breaks`는 이미 의존성, iframe 렌더는 `react-dom/client` `createRoot` 사용
- 프린트 다이얼로그는 OS 레벨이라 완전 자동 저장 불가 — 사용자가 'PDF로 저장'을 직접 선택 (이번 scope에서는 의도된 동작)
- iframe은 same-origin이어야 `contentDocument`/`print()` 접근 가능 — `srcdoc` 또는 동적 document.write로 동일 출처 유지
- 마진 단위는 cm (에디터와 동일). `@page margin`에 `cm` 단위 그대로 사용
- `front-assets/`는 건드리지 않음
- Editor의 기존 props/저장 로직은 변경하지 않음 — 버튼 추가만

## Direction

- 트리거: `Editor.jsx` 상단 바(제목/분석/채팅 토글이 있는 영역)에 버튼 추가. onClick → `exportPdf(file)`
- iframe 전략: `position:fixed; width:0; height:0; border:0`로 화면 밖 iframe 생성, 부모와 same-origin. iframe `document`에 `<style>`(@page + 본문) 주입 후 `createRoot(iframeBody).render(<Markdown>{content}</Markdown>)`
- 마크다운 컴포넌트는 ReferencePane의 렌더 설정을 함수로 추출해 공유하거나, 동일 plugin 배열로 인라인 구성 — 뷰어와 출력 일관성 유지
- 인쇄 안정화: react 렌더는 동기적이지 않으므로 `requestAnimationFrame` 2틱 또는 `setTimeout(0)` 후 `print()` 호출. 폰트/레이아웃이 늦게 잡히면 `iframe.onload` 활용
- 정리: `contentWindow.onafterprint`에서 root.unmount() + iframe.remove(). 다이얼로그 취소 대비 일정 시간 뒤 fallback 정리
