# 텍스트 분석 기능 — 분석하기 버튼과 우측 분석 패널

편집기 본문을 챕터/씬/문단/대사/문장으로 파싱해 전체·챕터별 통계와 문장별 메타데이터를 우측 패널에 보여주는 분석 기능을 추가한다.

## Purpose

쓰는 사람이 작성 중인 원고의 구조와 분량을 한눈에 파악할 수 있도록 한다. 매번 새로 파싱하므로 본문 파일은 그대로 두면서, 챕터 수·문단 수·문장 수·글자 수 같은 기본 통계를 제시한다. 향후 추가될 문장 단위 메타데이터(가독성, 감정 등)의 토대를 만든다.

## Scope

**In scope**
- Tauri Rust 백엔드(`src-tauri`)에 palimpsest 파싱 규칙을 포팅한 분석 모듈 추가 — `# ` 챕터, `***`/`---`/`===` 씬 구분, blank-line 문단, `「…」`/`"…"` 대사, 그 외 narration, 문장 종결(`.`, `?`, `!`, `…`, 줄바꿈) 단위 분할
- `analyze_text` invoke 커맨드 노출 (입력: 본문 문자열, 출력: AST 요약 구조)
- 에디터 상단 툴바에 `분석하기` 아이콘 버튼 추가
- 우측 사이드 패널(분석 패널) — 전체 총계(챕터/씬/문단/문장/글자, 대사 비율)와 챕터별 요약(제목, 문단·문장 수) 표시
- 각 문장 노드에 글자 수 메타데이터(`charCount`) 부착 — 메모리에만 보관, 매 분석 시 재계산
- Rust 파서에 대한 단위 테스트(샘플 텍스트 픽스처 기반)

**Out of scope (later)**
- 문장 메타데이터의 영구 저장(.meta.json 사이드카, 본문 인라인 마커)
- 가독성·감정 등 추가 메타데이터
- diff/restore 등 palimpsest 의 다른 명령
- 문단별 트리 드릴다운 UI, 인터랙티브 검색
- 외부 도구로의 export

## Stages

1. Rust 파서 + invoke — `src-tauri` 에 분석 모듈과 `analyze_text` 커맨드, 단위 테스트까지 포함해 백엔드만으로 동작 검증
2. UI — 에디터 상단 툴바에 `분석하기` 버튼, 우측 분석 패널 컴포넌트, invoke 호출과 결과 렌더링 연결

## Constraints

- 새 npm/cargo 의존성은 최소화 — 정규식·문자열 처리는 표준 라이브러리 위주
- 기존 우측 패널(ReferencePane) UI 패턴과 토글 동작을 따른다 (AppShell 의 패널 상태 관리 흐름 활용)
- 본문 .md 파일 포맷은 변경하지 않는다
- 한국어 문장 종결(`다.`, `요.`, `…` 등)과 따옴표/괄호 안의 마침표 처리에서 오탐을 줄일 수 있는 휴리스틱 사용 — 완벽 해결은 추후 과제
- Semver: minor bump (`tauri.conf.json`, `src-tauri/Cargo.toml` 동시 갱신)

## Direction

Rust 쪽에서는 palimpsest 의 노드 트리 개념을 그대로 따와 `Document → Chapter → (Scene?) → Paragraph → (Dialogue|Narration) → Sentence` 형태의 AST 를 구성한다. 라인 단위로 토크나이즈 후 prefix 규칙(`#`, `***`/`---`/`===`)과 blank-line 으로 상위 구조를 만들고, 문단 안에서 따옴표/괄호로 대사 블록을 분리한 뒤 종결부호 기준으로 문장을 자른다. 각 문장 노드에 `charCount` 를 계산해 붙이고, 최상위에서 챕터별·전체 카운트를 집계해 반환한다.

프론트에서는 `analyze_text` 호출을 분석 패널 컴포넌트가 담당한다. 패널은 AppShell 의 우측 패널 슬롯을 재사용하고, 상단 툴바 버튼은 패널 토글 + 즉시 재분석 트리거 역할을 한다. 결과는 카드형 전체 통계 위에 챕터별 요약 리스트가 따라오는 평면 레이아웃으로 표시한다.
